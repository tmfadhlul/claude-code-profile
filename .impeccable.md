## Design Context

### Users
Individual developers and small engineering teams managing multiple Claude Code and Codex accounts from a local machine. They open dashboard during setup, account switching, drift repair, session recovery, secret rotation, and cross-device replication. Primary job: understand current state quickly, make safe changes with minimal ceremony, and always know what happens next.

### Brand Personality
Calm, precise, crafted. Interface should feel like trustworthy operator console built by someone who understands terminal workflows: confident without aggression, technical without jargon overload, warm without becoming playful.

### Aesthetic Direction
Flagship-quality “operator’s field notebook”: warm paper-toned workspace, deep ink navigation, restrained vermilion for primary actions, moss for healthy state, amber for attention. Editorial display typography paired with highly legible sans-serif body. Light-first, tactile borders, deliberate asymmetry, compact data where needed. Avoid generic monochrome shadcn dashboard, glassmorphism, neon developer styling, gradients, card grids, and excessive monospace.

### Design Principles
1. State before controls: show health, drift, and consequences before asking for action.
2. Progressive disclosure: common path stays obvious; advanced env, provider, link, and transport details open only when needed.
3. Safe operations feel calm: dangerous actions get explicit copy and visual separation; reversible behavior stays visible.
4. Preserve CLI mental model: names and commands match `clp`, but interface removes need to remember syntax.
5. Dense where comparison matters, spacious where decisions matter; mobile adapts actions without hiding them.
6. Every empty, loading, success, and failure state provides next useful action.
